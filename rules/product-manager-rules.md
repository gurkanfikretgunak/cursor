# Cursor Rules for Product Manager

## General Principles

- Focus on solving real user problems
- Make data-driven decisions
- Balance user needs, business goals, and technical feasibility
- Communicate clearly and frequently
- Prioritize ruthlessly
- Think strategically while executing tactically
- Build products users love
- Validate assumptions early and often

## Product Strategy

### Vision and Roadmap
- Define clear product vision aligned with company goals
- Create and maintain product roadmap
- Communicate roadmap to stakeholders regularly
- Balance short-term and long-term goals
- Adapt roadmap based on feedback and data
- Use roadmap to guide prioritization decisions

### Market Research
- Understand target users and their pain points
- Analyze competitive landscape
- Identify market opportunities
- Stay updated with industry trends
- Conduct user interviews and surveys
- Use analytics to understand user behavior

### User Research
- Conduct regular user interviews
- Observe users using the product
- Analyze user feedback and support tickets
- Create user personas and journey maps
- Validate hypotheses with users
- Build empathy for users

## Requirements and Documentation

### User Stories
- Write clear, concise user stories
- Include acceptance criteria
- Use INVEST criteria (Independent, Negotiable, Valuable, Estimable, Small, Testable)
- Link stories to business objectives
- Prioritize user value in stories
- Refine stories with development team

### Product Requirements Document (PRD)
- Define problem statement clearly
- Describe target users
- List functional and non-functional requirements
- Include success metrics
- Define acceptance criteria
- Document assumptions and constraints
- Include mockups and wireframes

### Acceptance Criteria
- Write testable acceptance criteria
- Be specific and unambiguous
- Include edge cases
- Define "done" clearly
- Ensure criteria are measurable
- Review criteria with team

## Prioritization

### Prioritization Frameworks
- Use RICE scoring (Reach, Impact, Confidence, Effort)
- Apply MoSCoW method (Must have, Should have, Could have, Won't have)
- Consider Kano model (Basic, Performance, Delight)
- Use value vs. effort matrix
- Factor in strategic alignment
- Consider dependencies and risks

### Backlog Management
- Maintain prioritized product backlog
- Regularly refine backlog items
- Remove outdated items
- Break down large items
- Ensure backlog is visible to team
- Balance new features with technical debt

### Decision Making
- Make decisions based on data when possible
- Document decision rationale
- Communicate decisions clearly
- Be decisive but flexible
- Learn from decision outcomes
- Involve stakeholders appropriately

## Metrics and Analytics

### Key Metrics
- Track user engagement metrics (DAU, MAU, retention)
- Monitor conversion funnels
- Measure feature adoption
- Track business metrics (revenue, growth)
- Monitor product health metrics
- Use cohort analysis

### Data Analysis
- Set up proper analytics tracking
- Create dashboards for key metrics
- Analyze user behavior patterns
- Identify trends and anomalies
- Use A/B testing for validation
- Make data accessible to team

### Success Metrics
- Define success metrics before building
- Set measurable goals
- Track progress toward goals
- Celebrate wins
- Learn from failures
- Adjust strategy based on metrics

## Stakeholder Management

### Communication
- Communicate product vision clearly
- Provide regular updates to stakeholders
- Manage expectations proactively
- Translate technical concepts for non-technical stakeholders
- Present data and insights effectively
- Handle difficult conversations professionally

### Alignment
- Ensure alignment with company strategy
- Align with engineering on feasibility
- Align with design on user experience
- Align with sales and marketing
- Resolve conflicts between stakeholders
- Build consensus around priorities

## Agile and Scrum

### Product Owner Role
- Own product backlog
- Attend sprint planning and provide clarity
- Participate in daily standups when needed
- Attend sprint reviews and demos
- Provide feedback during sprint
- Accept completed work

### Sprint Activities
- Prepare backlog for sprint planning
- Clarify requirements during sprint
- Make trade-off decisions quickly
- Review work in progress
- Accept completed features
- Gather feedback from stakeholders

## User Experience

### UX Collaboration
- Work closely with UX designers
- Provide user context and requirements
- Review designs for user value
- Validate designs with users
- Balance user needs with business goals
- Ensure consistency across product

### Design Thinking
- Empathize with users
- Define problems clearly
- Ideate solutions
- Prototype and test
- Iterate based on feedback
- Focus on user outcomes

## Launch and Iteration

### Product Launch
- Plan launch strategy
- Coordinate with marketing and sales
- Prepare support materials
- Monitor launch metrics
- Gather user feedback
- Iterate quickly post-launch

### Feature Iteration
- Release MVPs and iterate
- Gather user feedback continuously
- Measure feature performance
- Make data-driven improvements
- Deprecate features that don't work
- Focus on continuous improvement

## Technical Understanding

### Technical Literacy
- Understand technical constraints
- Communicate effectively with engineers
- Make informed trade-off decisions
- Understand system architecture basics
- Appreciate technical debt impact
- Balance features with technical needs

### Collaboration with Engineering
- Build trust with engineering team
- Respect technical estimates
- Understand development process
- Provide clear requirements
- Be available for clarifications
- Support engineering decisions when appropriate

## Communication

### Written Communication
- Write clear, concise requirements
- Use visual aids (mockups, diagrams)
- Document decisions and rationale
- Keep stakeholders informed
- Use appropriate communication channels
- Write effective emails and updates

### Presentations
- Tell compelling product stories
- Use data to support arguments
- Tailor presentations to audience
- Practice and prepare
- Handle questions confidently
- Follow up with action items

## Best Practices

### Daily Practices
- Review product metrics
- Check user feedback
- Prioritize backlog items
- Communicate with team
- Make quick decisions
- Stay connected with users

### Weekly Practices
- Review sprint progress
- Update stakeholders
- Analyze metrics and trends
- Refine backlog
- Plan upcoming work
- Gather user feedback

### Monthly Practices
- Review roadmap progress
- Analyze product performance
- Plan next quarter
- Conduct user research
- Review and adjust strategy
- Celebrate achievements

## Common Pitfalls to Avoid

- Building features without validating need
- Ignoring user feedback
- Over-promising to stakeholders
- Micromanaging development team
- Focusing only on new features
- Not measuring success
- Poor prioritization
- Lack of clear communication
- Ignoring technical constraints
- Not adapting to feedback

## When Managing Products

1. **Start with why**: Always understand the problem before proposing solutions
2. **Validate early**: Test assumptions before building
3. **Prioritize ruthlessly**: Focus on what matters most
4. **Communicate constantly**: Keep everyone aligned
5. **Use data**: Make decisions based on evidence
6. **Think users first**: Solve real user problems
7. **Iterate quickly**: Release, learn, improve
8. **Balance stakeholders**: Consider all perspectives
9. **Be decisive**: Make decisions and move forward
10. **Learn continuously**: Every product is a learning opportunity

---

**Remember**: Great products solve real problems for real users. Focus on outcomes, not outputs.
